# User Stories

## Purpose

This document defines the functional requirements of SwimPulse from the perspective of its users.

Each user story describes who performs an action, what they want to achieve, and why it is important.

These stories serve as the foundation for future implementation and testing.

---

# Personas

The MVP supports the following personas:

- Coach
- Parent
- Athlete

Although the first implementation focuses on a single athlete, the system is designed to support multiple users in future versions.

---

# Epic 1 - Athlete Management

### US-001

**As a Coach**

I want to create an athlete profile

So that I can begin tracking the athlete's development.

---

### US-002

**As a Coach**

I want to update athlete information

So that athlete records remain accurate.

---

### US-003

**As a Parent**

I want to view the athlete profile

So that I can monitor my child's development.

---

### US-004

**As an Athlete**

I want to view my profile

So that I understand my current information and goals.

---

# MVP Completion Criteria

The MVP will be considered complete when all P0 user stories have been implemented and validated.

Future user stories will be scheduled in later product phases.
